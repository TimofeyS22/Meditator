# Device-Level A11y Audit (VoiceOver/TalkBack)

Этот документ фиксирует реальный сценарный аудит для мобильного устройства и критерии выхода на уровень 10/10.

## Подготовка

1. iOS: включить VoiceOver, Reduce Motion, Larger Text (максимум +2 шага).
2. Android: включить TalkBack, Remove animations, Display size/Text size +1..+2.
3. Пройти все флоу только жестами скрин-ридера (без тапа по координатам).
4. В каждом флоу проверять:
   - корректные имена элементов;
   - логичный фокус-порядок;
   - наличие состояния (selected, enabled, loading);
   - отсутствие “немых” контролов;
   - отсутствие блокирующих визуальных анимаций при Reduce Motion.

## Критичные флоу (12)

### 1) Onboarding Welcome -> Goals -> Stress -> Prefs -> Finish
- Ожидание: кнопки озвучиваются, выбранные карточки имеют состояние selected.
- Критерий PASS: можно завершить онбординг без зрительного контроля.

### 2) Home -> Quick Actions
- Ожидание: карточки быстрых действий озвучиваются как кнопки.
- Критерий PASS: можно попасть в Library/Breathe/Journal/Pair через скрин-ридер.

### 3) Home -> Library -> Filter -> Open meditation
- Ожидание: фильтры категорий имеют selected/not selected.
- Критерий PASS: карточка медитации читает название и длительность.

### 4) Library -> Aura chat
- Ожидание: FAB имеет понятный label.
- Критерий PASS: можно открыть чат и отправить сообщение.

### 5) Aura chat -> Create meditation -> Player
- Ожидание: send/close/create action корректно озвучиваются.
- Критерий PASS: нет “безымянных” кнопок.

### 6) Player full control
- Ожидание: play/pause/stop/seek/speed/ambient доступны скрин-ридеру.
- Критерий PASS: возможно управление сессией без визуального контроля.

### 7) Player completion overlay -> Garden
- Ожидание: CTA озвучены, фокус не теряется.
- Критерий PASS: можно закрыть оверлей и перейти в сад.

### 8) Journal -> New entry (4 steps) -> Save
- Ожидание: прогресс шагов озвучивается как “Шаг N из 4”.
- Критерий PASS: интенсивность 1..5 выбирается и читается корректно.

### 9) Journal -> Analytics
- Ожидание: недельные точки имеют смысловые label (эмоция/нет записи).
- Критерий PASS: экран читаем и понятен по структуре.

### 10) Pair -> Messages -> Actions
- Ожидание: кнопки “подбодрить/напомнить/медитировать вместе” озвучены.
- Критерий PASS: сообщение отправляется без потери фокуса.

### 11) Find partner -> Copy invite -> Share
- Ожидание: invite code имеет явный label и действие копирования.
- Критерий PASS: код копируется, share запускается.

### 12) Profile -> Settings -> Back -> Delete account dialog
- Ожидание: секции настроек и destructive action имеют понятные labels.
- Критерий PASS: диалог удаления полностью управляем скрин-ридером.

## Известные риски, требующие device-проверки

- Ручной фокус-порядок внутри complex overlays.
- Поведение озвучки на кастомно отрисованных элементах (CustomPaint) в разных версиях ОС.
- Избыточные анимации shimmer/scale в отдельных сценариях (при нестабильном FPS).

## Критерии 10/10 (Definition of Done)

1. 100% элементов в 12 флоу имеют корректные labels/roles/states.
2. 0 блокеров по VoiceOver/TalkBack (невозможно выполнить действие).
3. Reduce Motion полностью выключает декоративную динамику, не ломая UX.
4. Контраст текста >= WCAG AA для body/small text на всех ключевых экранах.
5. Ни один critical flow не требует зрительного контроля.
6. Фокус-порядок совпадает с визуальной иерархией.
7. Ошибки/лоадинг/успех всегда озвучиваются и имеют понятные тексты.

## Протокол фиксации результатов

Для каждого флоу:
- Статус: PASS / FAIL
- Платформа: iOS / Android
- Девайс и ОС
- Шаг воспроизведения
- Фактическая озвучка
- Ожидаемая озвучка
- Severity: Blocker / Major / Minor
- Ссылка на задачу фикса
